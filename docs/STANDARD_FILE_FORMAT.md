# Standard File Format

Every `.md` file in the system should adhere to this standard format to ensure consistency across the Maroon OS.

```markdown
# Title

## Summary
Short, high-level description.

## Purpose
Why this file exists.

## Architecture
How it fits into the system.

## Components
List of sub-components.

## Data Flows
How data moves in/out.

## Agent Interactions
Which agents use this.

## Skills Required
Which skills this file depends on.

## Terraform Integration
How infra is deployed.

## BigQuery Integration
How data is stored.

## Free-Tier Usage
Which free tiers power this.

## Credit Usage
Which credits power this.

## Security
IAM, isolation, sandboxing.

## Future Expansion
What comes next.
```\n
